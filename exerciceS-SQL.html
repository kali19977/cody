SQL

EXERCICE 1 :

Considerons une table 'Clients' contient les colonnes : nom,pays,adresse.

Afficher tous les enregistrements de la table "Clients".
SOLUTION :
SELECT * FROM Clients;
EXPLICATION :
Cette requête sélectionne toutes les lignes de la table "Clients" et les affiche.

EXERCICE 2 :

Afficher les noms et les adresses des clients dont le pays est "France".
SOLUTION :
SELECT nom, adresse FROM Clients WHERE pays = 'France';
EXPLICATION :
Cette requête sélectionne les noms et les adresses des clients dont le pays est "France".

EXERCICE 3 :

Considerons une table "commandes" contient les colonnes : id,quantite,adresse

Afficher le nombre total de commandes dans la table "Commandes".
SOLUTION :
SELECT COUNT(*) AS total_commandes FROM Commandes;
EXPLICATION :
Cette requête utilise la fonction COUNT() pour compter le nombre total de lignes dans la table "Commandes".

EXERCICE 4 :

Considerons une table 'produits' contient les colonnes : Prix,Id et nom.

Afficher les produits dont le prix est supérieur à 100 euros.
SOLUTION :
SELECT * FROM Produits WHERE prix > 100;
EXPLICATION :
Cette requête sélectionne tous les enregistrements de la table "Produits" où le prix est supérieur à 100.

EXERCICE 5 :

Insérer un nouveau client dans la table "Clients" avec les valeurs suivantes : nom = "Jean Dupont", adresse = "10 rue des Fleurs", pays = "France".
SOLUTION :
INSERT INTO Clients (nom, adresse, pays) VALUES ('Jean Dupont', '10 rue des Fleurs', 'France');
EXPLICATION :
Cette requête insère un nouvel enregistrement dans la table "Clients" avec les valeurs spécifiées.

EXERCICE 6:

Mettre à jour le prix du produit ayant l'ID 123 à 200 euros.
SOLUTION :
UPDATE Produits SET prix = 200 WHERE ID = 123;
EXPLICATION :
Cette requête met à jour le prix du produit ayant l'ID 123 avec une nouvelle valeur de 200.

EXERCICE 7:

Supprimer tous les enregistrements de la table "Commandes" où la quantité est inférieure à 10.
SOLUTION :
DELETE FROM Commandes WHERE quantite < 10;
EXPLICATION :
Cette requête supprime tous les enregistrements de la table "Commandes" où la quantité est inférieure à 10.

EXERCICE 8:

Afficher le nombre de clients par pays dans la table "Clients".
SOLUTION :
SELECT pays, COUNT(*) AS nombre_clients FROM Clients GROUP BY pays;
EXPLICATION :
Cette requête utilise la clause GROUP BY pour regrouper les clients par pays et la fonction COUNT() pour compter le nombre de clients par pays.

EXERCICE 9:

Afficher les produits dont le nom contient le mot "chaussures".
SOLUTION :
SELECT * FROM Produits WHERE nom LIKE '%chaussures%';
EXPLICATION :
Cette requête sélectionne tous les produits de la table "Produits" dont le nom contient le mot "chaussures" en utilisant le mot-clé LIKE et le caractère générique "%".

EXERCICE 10:

créer une table appelée "Students" avec des colonnes pour l'identifiant de l'étudiant (entier), le nom (texte) et l'âge (entier)
SOLUTION :
CREATE TABLE Students (
student_id INT,
name TEXT,
age INT
);
EXPLICATION :
Cette requête crée une table nommée "Students" avec trois colonnes : "student_id" de type INTEGER, "name" de type TEXT et "age" de type INTEGER.

EXERCICE 11:

créer une table appelée "Books" avec des colonnes pour l'identifiant du livre (entier), le titre (texte), l'auteur (texte) et l'année de publication (entier)
CREATE TABLE Books (
book_id INT,
title TEXT,
author TEXT,
year INT
);
EXPLICATION :
Cette requête crée une table nommée "Books" avec quatre colonnes : "book_id" de type INTEGER, "title" de type TEXT, "author" de type TEXT et "year" de type INTEGER.

EXERCICE 12:

Considérez une table appelée "Étudiants" avec des colonnes pour l'identifiant de l'étudiant, le nom et la note. Récupérez les noms et les notes des étudiants dans l'ordre croissant des notes, en excluant les étudiants ayant une note inférieure à 70.
SELECT nom, note
FROM Étudiants
WHERE note >= 70
ORDER BY note ASC;
EXPLICATION :
Cette requête sélectionne les noms et les notes des étudiants de la table Étudiants où la note est supérieure ou égale à 70. Les résultats sont ensuite triés par ordre croissant des notes en utilisant la clause ORDER BY.

EXERCICE 13:

Considérez une table appelée "Orders" avec des colonnes pour l'ID de commande, l'ID de client et le montant total. Récupérez les IDs de commande et les montants totaux des commandes où le montant total est supérieur à 1000 $ et l'ID de client n'est pas nul.
SELECT order_id, total_amount
FROM Orders
WHERE total_amount > 1000 AND customer_id IS NOT NULL;
EXPLICATION :
Cette requête sélectionne les colonnes "order_id" et "total_amount" de la table "Orders" où la valeur de la colonne "total_amount" est supérieure à 1000 et la valeur de la colonne "customer_id" n'est pas nulle (non null).

Cela permet de récupérer les IDs de commande et les montants totaux des commandes qui satisfont les conditions spécifiées, à savoir un montant total supérieur à 1000 $ et un ID de client non nul.

EXERCICE 14:

Considérons deux tables : "Students" avec les colonnes student_id, student_name et "Grades" avec les colonnes student_id et grade. Les deux tables sont liées par la colonne student_id.

Enoncé : Écrivez une requête SQL pour récupérer les noms des étudiants ayant obtenu une note supérieure à 90.
SELECT student_name
FROM Students
JOIN Grades ON Students.student_id = Grades.student_id
WHERE grade > 90;
EXPLICATION :
La requête utilise la clause JOIN pour combiner les deux tables "Students" et "Grades" en utilisant la colonne student_id comme condition de jointure.

La clause WHERE est utilisée pour filtrer les résultats en ne sélectionnant que les lignes où la note (grade) est supérieure à 90.

EXERCICE 15:

Considérons deux tables : "Orders" avec les colonnes order_id, customer_id et total_amount, et "Customers" avec les colonnes customer_id et customer_name. Les deux tables sont liées par la colonne customer_id.

Enoncé : Écrivez une requête SQL pour récupérer les informations des commandes (order_id, total_amount) avec les noms des clients correspondants.
SELECT Orders.order_id, Orders.total_amount, Customers.customer_name
FROM Orders
JOIN Customers ON Orders.customer_id = Customers.customer_id;
EXPLICATION :
La requête utilise la clause JOIN pour combiner les deux tables "Orders" et "Customers" en utilisant la colonne customer_id comme condition de jointure.

La requête SELECT est utilisée pour récupérer les colonnes order_id et total_amount de la table "Orders", ainsi que la colonne customer_name de la table "Customers".

Les résultats afficheront les informations des commandes avec les noms des clients correspondants.
